# 囲碁ルール仕様書（日本ルール）

本仕様書は日本棋院・関西棋院が定める公式日本ルールを元に、ソフトウェア実装に必要な情報をまとめたものです。

---

## 1. 盤面とコンポーネント

### 碁盤

| 種別 | 路数 | 交点数 |
|------|------|--------|
| 標準 | 19×19 | 361 |
| 中型 | 13×13 | 169 |
| 小型 | 9×9  | 81    |

- 盤面は縦横の線が交差した**交点**で構成される
- 全交点は対局開始時に空（empty）

### 星（ほし）

19×19 盤の場合、9つの星が以下の座標に存在する（1-indexed、左上を(1,1)とする）:

```
(4,4)  (4,10)  (4,16)
(10,4) (10,10) (10,16)
(16,4) (16,10) (16,16)
```

ハンデ石の配置基準点として使用。

### 石

- 色：黒（Black）・白（White）の2種
- 一度置かれた石は**移動不可**
- アゲハマ（取られた石）のみ盤上から除去される

---

## 2. ゲームの進行

### 手番

- **黒が先手**（先番）
- 黒・白が交互に着手する
- 1手番につき1つの交点に1石を置く、または**パス**を宣言する

### 合法手の条件

以下をすべて満たす交点への着手が合法:

1. **空点であること**（既に石が置かれていない）
2. **自殺手でないこと**（後述）
3. **コウのルールに違反しないこと**（後述）

---

## 3. 連（れん）と呼吸点（ダメ）

### 連（グループ）

- 同色の石が**縦横に隣接**して繋がったひとまとまりを「連」と呼ぶ
- 斜めは連結とみなさない
- 連は捕獲・ダメの計算において1つの単位として扱う

### 呼吸点（ダメ / Liberty）

- ある連に縦横隣接する**空点**の数
- 連のダメが0になると即時に捕獲される

```
例: 盤端の石は最大3ダメ、角は最大2ダメ
```

### アタリ

- 連のダメが残り1の状態をアタリと呼ぶ
- 次の手で相手に取られる危険がある状態

---

## 4. 捕獲（アゲハマ）

### 捕獲のタイミング

着手後、**相手の連**のダメが0になった場合、その連を即時に盤上から除去する。

### 処理順序（重要）

1. 石を置く
2. **相手の連**のダメをチェックし、0なら除去（アゲハマとして記録）
3. その後、**自分の連**のダメをチェック

この順序により、相手石を取ることで自分の連のダメが生じる場合の着手は合法となる。

### 複数グループの同時捕獲

1手で複数の相手グループを同時に取ることができる。

---

## 5. 自殺手の禁止

- 着手後に**相手石を1つも取らず**、かつ**自分の連のダメが0になる**着手は禁止（自殺手）
- 相手石を取ることでダメが生じる場合は合法

### 判定ロジック

```
is_suicide(point, color):
  仮置きして相手石を取る
  取った結果、自分の連のダメ = 0 かつ 相手石を取っていない → 禁手
  それ以外 → 合法
```

---

## 6. コウのルール

### 基本コウ

- 直前の着手で相手が取った1石を、**直後の手番で同じ交点に置いて**盤面を1手前と同じ状態に戻すことを禁止する
- これを「コウ」と呼ぶ

### 実装上の判定

```
着手後の盤面状態 = 直前の着手前の盤面状態 → 禁手（コウ違反）
```

1手前の盤面状態を保持して比較する。

### コウ争い

- コウになった側はまず「コウ立て」（盤上の他の場所への着手）を行う
- 相手がコウ立てに応じたら、コウを取り返すことができる

### 劫（サイクル）と無勝負

- 単純なコウを超えた複雑な繰り返し（同一盤面の3回以上の出現など）が発生した場合、**無勝負**（引き分けではなく、ゲーム無効）となる
- 日本ルールではスーパーコウ（全盤面状態の繰り返し禁止）は**採用しない**

---

## 7. ゲームの終了

### 終局条件

**両者が連続してパス**した時点でゲーム終了。

```
pass_count >= 2 → 終局
```

### 死石の合意

終局後、両者で死んでいる石（取り除くべき石）を合意によって決める。

- 死石 = 実戦を続けた場合に取られることが明らかな連
- 合意した死石は相手のアゲハマに加える

### 争い（死活論争）

死石について合意できない場合、その箇所から対局を再開して決着をつける。
再び両者パスで終局したとき、盤上に残っている石は「生き」とみなす。

---

## 8. 計算（日本式地計算）

### 計算式

```
スコア = 地（囲った空点） + アゲハマ（取った石） + コミ（白のみ）
```

### 地（じ / Territory）

- 自分の**生きている石**だけに囲まれた空点の数
- セキの空点は**どちらの地にもカウントしない**

### アゲハマ（Prisoners）

- 対局中に取った相手石
- 終局後の死石も含む

### 実際の数え方

1. 死石を除去し、アゲハマに加える
2. アゲハマを相手の地に置いて埋める
3. 残った空点を数える（整地）

### コミ（Komi）

| 対局種別 | コミ |
|----------|------|
| 互先（even game）| 白に6.5目 |
| ハンデ戦 | 白に0.5目 |

- 0.5目が含まれることで引き分けが発生しない
- 日本棋院は2002年9月に6.5目を正式採用

---

## 9. セキ（相互活き）

### 定義

- 対局する2グループが互いに相手を取れない状態で共存する局面
- 共有のダメに打つと自分のグループが取られる危険があるため、両者とも打てない

### セキの地の扱い

- セキの中の空点（共有ダメ）は**どちらの地にも数えない**（中立点 / ダメ）
- セキにある石は「生き」として盤上に残る

---

## 10. 死活（Life and Death）

### 生きている石（活き石）

次の条件のいずれかを満たすグループは「生き」:

- **2つの独立した目（め）を持つ**
- セキによる共存状態にある

### 目（め / Eye）

- グループに完全に囲まれた空点
- 独立した2つの目があれば、相手はどちらにも打てないため取られない

### 死んでいる石（死石）

- 目が2つなく、実戦を続けると取られることが明らかな石
- 終局後に相手が取り除いてアゲハマに加える

### 曲四（マガリ四目）

- 角に形成される特定の4石の形で、2目を作れない形
- 日本ルールでは実戦不要で**死石**として扱う（相手の申告で死石となる）

---

## 11. ハンデ戦（置き碁）

### ハンデ石の数と配置（19×19）

| 置き数 | 配置する星 |
|--------|-----------|
| 2石 | 右下、左上 |
| 3石 | 右下、左上、右上 |
| 4石 | 4隅すべて |
| 5石 | 4隅 + 天元 |
| 6石 | 4隅 + 左右辺中央 |
| 7石 | 4隅 + 左右辺中央 + 上辺中央 |
| 8石 | 4隅 + 左右辺中央 + 上下辺中央 |
| 9石 | 9星すべて |

具体的な座標（row, col、1-indexed）:

```
天元（center）: (10, 10)
隅（corners）:  (4,4), (4,16), (16,4), (16,16)
辺中央（sides）: (10,4), (10,16), (4,10), (16,10)
```

### ハンデ戦の手順

1. 黒が指定された星にハンデ石を全て置く
2. **白が最初の一手**を打つ
3. 以後、黒・白と交互に着手

## 12. 実装仕様まとめ

### データ構造

```typescript
type Color = "black" | "white" | "empty";

interface BoardState {
  grid: Color[][];       // [row][col]
  prisoners: { black: number; white: number };
  currentTurn: "black" | "white";
  passCount: number;
  previousGrid: Color[][] | null; // コウ判定用（1手前の盤面）
  komi: number;
}
```

### 必須の処理

| 処理 | 概要 |
|------|------|
| `isLegal(point, color)` | 空点チェック・自殺手チェック・コウチェック |
| `placeStone(point, color)` | 石を置く→相手石取る→自石のダメ確認 |
| `findGroup(point)` | 連結する同色石を全て検索（BFS/DFS） |
| `countLiberties(group)` | 連の呼吸点数を返す |
| `removeGroup(group)` | 連を除去しアゲハマに加算 |
| `calculateScore()` | 地・アゲハマ・コミを合計してスコアを返す |
| `identifyTerritory()` | 各空領域が黒・白・中立のどれかを判定 |

### コウ判定の実装

```
着手 → 仮盤面生成 → 仮盤面 === 直前盤面 → 禁手
```

1手前の盤面（`previousGrid`）のみ保持すればよい（日本ルールは単純コウのみ）。

### スコア計算手順

1. 死石を除去（プレイヤー合意 or AIによる判定）
2. 各空領域の帰属を判定（片方の色のみに接していれば、その色の地）
3. セキ内の空点は中立（ノーカウント）
4. 地 + アゲハマ（対局中） + アゲハマ（死石）を合計
5. 白にコミを加算
6. スコア大きい方が勝ち
